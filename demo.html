Frontend y backend

Arquitectura

Hay dos repos que se despliegan por separado. El frontend no toca la base de datos: cada lectura o escritura pasa por la API REST del backend, que es el único que habla con PostgreSQL.

dnd-campaign-hub-frontend

react + vite · port 5173
  • Renderiza las pantallas que ves abajo
  • Guarda el JWT en localStorage
  • Sin router: navegación por estado
  • services/api.js concentra los fetch
→

dnd-campaign-hub-backend

express + pg · port 3001
  • Valida el JWT en rutas protegidas
  • Middleware dmOnly para roles
  • Campañas, misiones, encuentros y usuarios
  • Tira iniciativa y lleva el estado del combate
→

PostgreSQL & Open5e

datos
  • PostgreSQL: usuarios, campañas, misiones
  • Encuentros y orden de iniciativa
  • Open5e aporta el catálogo de monstruos
  • Se consulta al vuelo, no se guarda en disco

Repos

Frontend

dnd-campaign-hub-frontend

SPA en React 18 con Vite. Solo habla con la API del backend.

  • npm install
  • npm run dev — arranca en localhost:5173
  • Necesita el backend en localhost:3001
Backend

dnd-campaign-hub-backend

API Express con PostgreSQL y auth por JWT.

  • npm install
  • Configurar .env con la base de datos
  • npm start — arranca en localhost:3001
Esta página es estática a propósito: vive en el repo del frontend y se publica con GitHub Pages, así se puede compartir sin desplegar backend ni base de datos. Los detalles están en el README de cada repo.
